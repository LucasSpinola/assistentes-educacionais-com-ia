# Papers

Espaço para anotações e resumos de papers lidos ao longo da trilha. A ideia é
registrar o entendimento de cada trabalho, não armazenar os PDFs, que devem ser
obtidos nas fontes oficiais.

## Notas disponíveis

- [Attention Is All You Need (Vaswani et al., 2017)](vaswani2017-attention.md), a base dos
  Transformers e dos LLMs.
- [Retrieval-Augmented Generation (Lewis et al., 2020)](lewis2020-rag.md), a base do RAG e do Tutor
  do projeto final.

## Como organizar

Crie um arquivo Markdown por paper, com um nome que ajude a localizar, por exemplo
`vaswani2017-attention.md`. Um resumo útil costuma ter:

- Referência completa e link, alinhados com `references/referencias.bib`.
- Problema que o trabalho ataca.
- Ideia central e o que há de novo.
- Método em linhas gerais.
- Resultados principais.
- Limitações e pontos de atenção.
- Como o trabalho se conecta com os módulos da trilha.

## Por que registrar leituras

Para a iniciação científica e o mestrado, manter resumos próprios acelera a
revisão de literatura e ajuda a enxergar conexões entre trabalhos. Cada resumo
deve apontar para a entrada correspondente na bibliografia.
