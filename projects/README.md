# Projetos

Projetos completos e executáveis, ligados aos módulos a partir do M9. Diferente
dos [notebooks](../notebooks/), que são didáticos e interativos, os projetos têm a
estrutura de uma pequena aplicação, com código organizado, README de execução e,
quando fizer sentido, testes.

## Projetos por módulo

| Módulo | Pasta | Projeto |
|---|---|---|
| 09 | [m09-rag-assistant](m09-rag-assistant/) | Assistente sobre documentos com RAG |
| 10 | [m10-tutor-agent](m10-tutor-agent/) | Agente tutor com ferramentas |
| 11 | [m11-multi-agent](m11-multi-agent/) | Time de agentes especializados |
| 12 | [m12-analytics-dashboard](m12-analytics-dashboard/) | Dashboard de Learning Analytics |
| 13 | [m13-student-modeling](m13-student-modeling/) | Sistema adaptativo de aprendizagem |
| 14 | [m14-final-assistant](m14-final-assistant/) | Assistente educacional multi-agente completo |

## Padrão de cada projeto

Cada pasta de projeto traz um `README.md` com o objetivo, as instruções de
execução e os requisitos, o código organizado em módulos Python, e um teste
mínimo quando aplicável. Os projetos rodam do zero, só com a biblioteca padrão, e
têm caminhos opcionais com bibliotecas e modelos locais via Ollama.
